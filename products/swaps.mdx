Swaps convert **EURC ↔ USDC on Solana**. Remitflex never holds deposit keys: you send the origin token to a deposit address, and the output token is delivered to your Solana wallet.

Every swap is tied to a **customer** under your organisation. Omit `customerId` to use your org's primary customer (created at signup).

**Required scopes:** `fx:read`, `transfers:read`, `transfers:write`

**Supported pair:** Solana `USDC` ↔ `EURC` only (both directions).

## When to use swaps

- Same-chain EURC/USDC conversion on Solana
- Treasury FX without operating a custodial hot wallet
- Indicative quotes via [`GET /v1/rates`](/products/rates) before create

<Tip>
  Need to **collect** into Solana EURC or USDC from another chain? Use a [payment route](/products/payment-routes) or [payment link](/products/payment-links). Use **swaps** when you already hold one side and want a treasury conversion with rates + deposit lifecycle.
</Tip>

For NGN/cNGN wallet flows, use [cNGN](/products/cngn).

## Lifecycle

| Status | Meaning |
|--------|---------|
| `awaiting_deposit` | Waiting for deposit to `depositAddress` |
| `deposit_received` | Origin deposit detected |
| `swapping` | Conversion in progress |
| `completed` | Output sent to `destinationAddress` |
| `refunded` | Origin deposit refunded |
| `failed` / `expired` / `cancelled` | Terminal states |

## Rates (indicative)

`GET /v1/rates` and `GET /v1/rates/{from}/{to}` return **live quotes**. Optional query params:

| Param | Description |
|-------|-------------|
| `amount` | Probe amount (human decimal). Default `1` |
| `tradeType` | `exact_input` (default) or `exact_output` |

Rates are indicative only. `POST /v1/swaps` always creates a **fresh** quote.

## Create a swap

```bash
curl -X POST https://api.remitflex.io/v1/swaps \
  -H "Authorization: Bearer $REMITFLEX_API_KEY" \
  -H "Idempotency-Key: $(uuidgen)" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "USDC",
    "to": "EURC",
    "amount": 100,
    "tradeType": "exact_input",
    "destinationAddress": "YOUR_SOLANA_WALLET",
    "refundTo": "YOUR_SOLANA_REFUND_WALLET",
    "customerId": "optional-customer-uuid"
  }'
```

| Field | Description |
|-------|-------------|
| `from` / `to` | `USDC` or `EURC` (must differ) |
| `amount` | Human-readable amount: send amount for `exact_input`, receive amount for `exact_output` |
| `tradeType` | `exact_input` (default) or `exact_output` |
| `destinationAddress` | Solana address receiving output tokens (**required**) |
| `refundTo` | Solana address for refunds (**required**) |
| `customerId` | Optional. Defaults to your org's primary customer |

Send the `from` token to `depositAddress` before `expiresAt`. Track with `GET /v1/swaps/{id}` (syncs status on read).

## List swaps by customer

```bash
curl "https://api.remitflex.io/v1/swaps?customerId=CUSTOMER_UUID" \
  -H "Authorization: Bearer $REMITFLEX_API_KEY"
```

## Track and cancel

```bash
# Status
curl https://api.remitflex.io/v1/swaps/{id} \
  -H "Authorization: Bearer $REMITFLEX_API_KEY"

# Cancel (only while awaiting_deposit)
curl -X POST https://api.remitflex.io/v1/swaps/{id}/cancel \
  -H "Authorization: Bearer $REMITFLEX_API_KEY" \
  -H "Idempotency-Key: $(uuidgen)"
```

## Failed / refunded swaps

When a swap reaches `failed` or `refunded`, check `errorMessage` and status fields. Surface these to your user and include the swap `id` when contacting Remitflex support.

The same fields appear on swap rows in [`GET /transactions`](/api-reference/list-transactions) under `receipt`.
