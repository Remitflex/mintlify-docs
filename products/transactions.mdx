**Scope:** `transfers:read`  
**Endpoint:** `GET /v1/transactions` (`?customerId=` optional)

One endpoint for reconciliation across Remitflex products. Returns up to 200 recent events, newest first (local records only).

| `kind` | Product |
|--------|---------|
| `payment_route` | Cross-chain route deposit |
| `swap` | Solana EURC↔USDC swap |
| `collection` | Payment link |
| `offramp` | Local fiat payout (`/v1/offramps`) |
| `onramp` | Local fiat deposit (`/v1/onramps`) |
| `cngn_convert` | cNGN wallet convert |
| `cngn_withdrawal` | cNGN wallet withdrawal |
| `cngn_payout` | cNGN → NGN bank payout |
| `cngn_payin` | Temporary VA payin |
| `cngn_deposit` | Permanent VA deposit |

Each row includes `status`, `amount`, `destination`, and `receipt`. Terminal statuses trigger a branded email to the linked customer and creating user.

## Filtering

```bash
curl "https://api.remitflex.io/v1/transactions?customerId=cus_abc123" \
  -H "Authorization: Bearer $REMITFLEX_API_KEY"
```

## Refreshing status

- Payment routes: `POST /v1/payment-routes/{id}/deposits/sync`
- Payment links: `POST /v1/collections/{id}/sync`
- Local fiat: `GET /v1/offramps/:orderId`, `GET /v1/onramps/:orderId?poll=true`
- cNGN: `GET /v1/cngn/converts/:id`, `/withdrawals/:id`, `/payouts/:id`, `/payins/:id`, `/deposits/:id`
