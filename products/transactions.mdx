**Scope:** `transfers:read`  
**Endpoint:** `GET /v1/transactions` (`?customerId=` optional)

One endpoint for reconciliation across every Remitflex product. Returns up to 200 recent events, newest first. Triggers a deposit sync before responding.

| `kind` | Product |
|--------|---------|
| `payment_route` | Cross-chain route deposit |
| `collection` | Payment link |
| `offramp` | Stablecoin → local fiat payout |
| `onramp` | Local fiat deposit → stablecoin delivery |

Each row includes `status`, `amount`, `destination`, and `receipt` (signatures, settled amounts, safe error messages).

## Filtering

```bash
curl "https://api.remitflex.io/v1/transactions?customerId=cus_abc123" \
  -H "Authorization: Bearer $REMITFLEX_API_KEY"
```

Use `customerId` to build per-user history across routes, collections, and fiat orders.

## Refreshing deposit status

For on-chain products, you can force a sync before reading the ledger:

- Payment routes: `POST /v1/payment-routes/{id}/deposits/sync`
- Payment links: `POST /v1/collections/{id}/sync`

For offramps and onramps, poll the order endpoint directly (`GET /v1/offramps/:orderId`, `GET /v1/onramps/:orderId?poll=true`).
