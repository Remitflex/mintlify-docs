Cross-chain pay-in: payer sends on one **network** (Bitcoin, Ethereum, Tron, etc.), you receive stablecoins on **Base** or **Solana**. Each route gets a persistent `depositAddress`.

**Scopes:** `collections:read`, `collections:write`

For hosted checkout, use [Payment links](/products/payment-links) instead.

Discover supported networks and corridors in [Networks & corridors](/concepts/networks-and-corridors) before creating a route.

## Endpoints

| Method | Path | Purpose |
|--------|------|---------|
| `POST` | `/payment-routes` | Create route |
| `GET` | `/payment-routes` | List routes |
| `GET` | `/payment-routes/{id}` | Get route |
| `GET` | `/payment-routes/deposits` | All deposits |
| `GET` | `/payment-routes/{id}/deposits` | Route deposits |
| `POST` | `/payment-routes/{id}/deposits/sync` | Refresh deposit state |
| `POST` | `/payment-routes/{id}/pause` | Pause route |
| `POST` | `/payment-routes/{id}/activate` | Activate route |
| `DELETE` | `/payment-routes/{id}` | Archive route |

## Create

Use network **keys** and token **codes** from [Networks & corridors](/concepts/networks-and-corridors) — not chain IDs or contract addresses:

```bash
curl -X POST https://api.remitflex.io/v1/payment-routes \
  -H "Authorization: Bearer $REMITFLEX_API_KEY" \
  -H "Idempotency-Key: $(uuidgen)" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Treasury inbound",
    "originChainKey": "tron",
    "destinationChainKey": "base",
    "destinationCurrency": "USDC",
    "destinationAddress": "0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb"
  }'
```

| Field | Description |
|-------|-------------|
| `originChainKey` | Where payers send (e.g. `bitcoin`, `ethereum`, `tron`, `base`) |
| `destinationChainKey` | Where you receive — `base` or `solana` |
| `destinationCurrency` | Token code — `USDC` or `USDT` |

`originChainKey` and `destinationChainKey` are required when creating routes or payment links. Remitflex resolves networks internally — customers never send or receive chain IDs.

Response includes `depositAddress` — share with payers on the origin network.

## Status

`active` → `paused` → `archived`. Deposits appear in [Transactions](/products/transactions) as `kind: payment_route`.
