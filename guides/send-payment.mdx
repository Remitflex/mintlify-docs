Payment routes let you collect funds on one chain and deliver them on another via Relay. This guide walks through creating a route and monitoring deposits.

## Prerequisites

- API key with `collections:read` and `collections:write` scopes
- Destination wallet address on the target chain

```bash
export API_BASE="https://api.remitflex.com/v1"
export REMITFLEX_API_KEY="rmf_test_YOUR_KEY"
```

## Step 1 — Explore corridors

```bash
curl -s "$API_BASE/payment-routes/corridors" \
  -H "Authorization: Bearer $REMITFLEX_API_KEY" | jq .
```

```bash
curl -s "$API_BASE/payment-routes/chains" \
  -H "Authorization: Bearer $REMITFLEX_API_KEY" | jq .
```

## Step 2 — Create a payment route

```bash
curl -s -X POST "$API_BASE/payment-routes" \
  -H "Authorization: Bearer $REMITFLEX_API_KEY" \
  -H "Idempotency-Key: $(uuidgen)" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Supplier payment",
    "originChainId": 792703809,
    "destinationChainId": 137,
    "destinationCurrency": "0x3c499c542cef5e3811e1192ce70d8cc03d5c3359",
    "destinationAddress": "0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb"
  }' | jq .
```

The response includes `depositAddress` — share this with the payer on the origin chain.

## Step 3 — Monitor deposits

```bash
ROUTE_ID="your-route-uuid"

curl -s "$API_BASE/payment-routes/$ROUTE_ID/deposits" \
  -H "Authorization: Bearer $REMITFLEX_API_KEY" | jq .
```

Force a sync from Relay:

```bash
curl -s -X POST "$API_BASE/payment-routes/$ROUTE_ID/deposits/sync" \
  -H "Authorization: Bearer $REMITFLEX_API_KEY" \
  -H "Idempotency-Key: $(uuidgen)" | jq .
```

## Step 4 — Unified ledger

All payment-route deposits and swaps appear in the transactions feed:

```bash
curl -s "$API_BASE/transactions" \
  -H "Authorization: Bearer $REMITFLEX_API_KEY" | jq '.data.transactions[:5]'
```

## Manage route lifecycle

```bash
# Pause
curl -X POST "$API_BASE/payment-routes/$ROUTE_ID/pause" \
  -H "Authorization: Bearer $REMITFLEX_API_KEY" \
  -H "Idempotency-Key: $(uuidgen)"

# Activate
curl -X POST "$API_BASE/payment-routes/$ROUTE_ID/activate" \
  -H "Authorization: Bearer $REMITFLEX_API_KEY" \
  -H "Idempotency-Key: $(uuidgen)"

# Archive
curl -X DELETE "$API_BASE/payment-routes/$ROUTE_ID" \
  -H "Authorization: Bearer $REMITFLEX_API_KEY" \
  -H "Idempotency-Key: $(uuidgen)"
```

See the [API reference](/api-reference/overview) for full schemas and error codes.
