RemitFlex supports two primary money-movement patterns today: **Solana swaps** (same-chain token conversion) and **payment routes** (cross-chain collection via Relay). Both appear in the unified `GET /v1/transactions` ledger.

## Solana swap flow

Swaps convert one Solana SPL token into another (e.g. USDC → EURC) via Jupiter.

| Status | Meaning |
|--------|---------|
| `awaiting_deposit` | Swap created; waiting for deposit to `depositAddress` |
| `deposit_received` | On-chain deposit detected |
| `swapping` | Jupiter swap executing |
| `completed` | Output delivered to `destinationAddress` |
| `failed` | Error during swap or payout |
| `expired` | Deposit window passed |
| `cancelled` | Cancelled while still `awaiting_deposit` |

<Steps>
  <Step title="Create swap">
    `POST /v1/swaps` with `from`, `to`, `amount`, and `destinationAddress`. Requires `transfers:write` and `Idempotency-Key`.
  </Step>
  <Step title="Deposit">
    Send the `from` token to the returned `depositAddress` (ATA) before `expiresAt`.
  </Step>
  <Step title="Settlement">
    RemitFlex detects the deposit, swaps via Jupiter, and pays out to your destination wallet.
  </Step>
  <Step title="Track">
    Poll `GET /v1/swaps/{id}` or check `GET /v1/transactions`.
  </Step>
</Steps>

See [Currency conversion](/guides/currency-conversion) for a full walkthrough.

## Payment route flow

Payment routes collect funds on an **origin chain** and deliver on a **destination chain** via Relay.

| Status | Meaning |
|--------|---------|
| `active` | Route accepts deposits |
| `paused` | Deposits paused |
| `archived` | Route removed |

<Steps>
  <Step title="Create route">
    `POST /v1/payment-routes` with origin/destination chain IDs, currencies, and destination address. Requires `collections:write`.
  </Step>
  <Step title="Share deposit address">
    Payers send funds to the route's `depositAddress` on the origin chain.
  </Step>
  <Step title="Relay settlement">
    Relay bridges and delivers to the destination address.
  </Step>
  <Step title="Track deposits">
    `GET /v1/payment-routes/{id}/deposits` or sync with `POST .../deposits/sync`.
  </Step>
</Steps>

See [Send payment](/guides/send-payment) for a full walkthrough.

## Choosing swaps vs payment routes

| Use case | Recommended |
|----------|-------------|
| Convert USDC → EURC on Solana | **Swap** |
| Collect on Ethereum, deliver on Polygon | **Payment route** |
| Same-chain Solana conversion | **Swap** |
| Multi-chain intake to one destination | **Payment route** |

## Authentication

- **API key** (`rmf_test_` / `rmf_live_`) — programmatic access with scopes
- **Dashboard JWT** — full access from the web app; no idempotency key required

Key management (`/v1/api-keys`) and login (`/v1/auth`) require JWT only.

## Status updates

Outbound webhooks are not yet available. Poll swap and deposit endpoints, or use `GET /v1/transactions` for a unified view.
