Use this endpoint to look up the current state of any off-ramp you have previously initiated. The response returns the complete off-ramp object, including the latest `status`, `updated_at` timestamp, and — if the off-ramp failed — a `failure_reason` field explaining what went wrong.

## Endpoint

```
GET https://api.remitflex.com/v1/offramp/{id}
```

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the off-ramp to retrieve. Off-ramp IDs are prefixed with `ofr_` and are returned when you [initiate an off-ramp](/api-reference/offramp/initiate).
</ParamField>

## Status Values

The `status` field on the off-ramp object reflects the current stage in the off-ramp lifecycle. The table below describes each possible value.

| Status | Description |
|---|---|
| `pending` | Off-ramp has been initiated and is queued for processing. |
| `processing` | RemitFlex is actively converting the stablecoin and preparing the bank transfer. |
| `settled` | Stablecoin conversion is complete and the bank transfer has been initiated. |
| `delivered` | The destination bank has confirmed receipt of the funds. |
| `failed` | The off-ramp could not be completed. Inspect the `failure_reason` field for the error code. |

## Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the off-ramp, prefixed with `ofr_`.
</ResponseField>

<ResponseField name="status" type="string">
  Current lifecycle state of the off-ramp. See [Status Values](#status-values) above.
</ResponseField>

<ResponseField name="amount" type="number">
  The stablecoin amount originally submitted for conversion.
</ResponseField>

<ResponseField name="source_currency" type="string">
  The stablecoin that was converted (e.g., `"USDC"`).
</ResponseField>

<ResponseField name="destination_currency" type="string">
  The ISO 4217 fiat currency code for the bank deposit (e.g., `"EUR"`).
</ResponseField>

<ResponseField name="converted_amount" type="number">
  The fiat amount deposited or to be deposited into the bank account, after fees.
</ResponseField>

<ResponseField name="fee" type="number">
  The total fee charged for this off-ramp, denominated in the `source_currency`.
</ResponseField>

<ResponseField name="bank_account" type="object">
  Bank account details associated with this off-ramp.

  <Expandable title="bank_account fields">
    <ResponseField name="account_holder_name" type="string">
      Legal name of the bank account holder.
    </ResponseField>

    <ResponseField name="account_number" type="string">
      IBAN or local account number on file for this off-ramp.
    </ResponseField>

    <ResponseField name="routing_number" type="string">
      ABA routing number, if applicable.
    </ResponseField>

    <ResponseField name="bank_name" type="string">
      Name of the receiving bank.
    </ResponseField>

    <ResponseField name="bank_country" type="string">
      ISO 3166-1 alpha-2 country code of the bank's domicile.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="reference" type="string">
  Your internal reference string, as provided when the off-ramp was created.
</ResponseField>

<ResponseField name="failure_reason" type="string">
  Present only when `status` is `failed`. Contains a machine-readable error code describing why the off-ramp could not be completed (e.g., `"invalid_account_number"`, `"compliance_rejected"`).
</ResponseField>

<ResponseField name="estimated_settlement" type="string">
  ISO 8601 timestamp of the expected settlement time. Only present while the off-ramp is in `pending` or `processing` state.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the off-ramp was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the most recent update to this off-ramp object.
</ResponseField>

<ResponseField name="metadata" type="object">
  The arbitrary key-value pairs attached to this off-ramp at creation time, returned as-is.
</ResponseField>

## Example Request & Response

<CodeGroup>

```bash curl
curl https://api.remitflex.com/v1/offramp/ofr_01HX9N4R \
  -H "Authorization: Bearer rf_live_xxxx"
```

```javascript JavaScript
const response = await fetch('https://api.remitflex.com/v1/offramp/ofr_01HX9N4R', {
  headers: {
    'Authorization': 'Bearer rf_live_xxxx',
  },
});

const offramp = await response.json();
console.log(offramp.status); // "settled"
```

## Polling vs. Webhooks

Polling this endpoint works for one-off lookups, but for production systems you should subscribe to webhook events instead. RemitFlex emits `offramp.settled` when the bank transfer is initiated and `offramp.delivered` when the destination bank confirms receipt — eliminating the need to poll and reducing unnecessary API calls.

See the [webhook events reference](/api-reference/webhooks/events) to explore all available event types.
