---
title: "POST /offramp: Initiate a Stablecoin Off-Ramp Transfer"
sidebarTitle: "Initiate Off-Ramp"
description: "Convert stablecoin to fiat and deposit funds directly to a bank account. Supports USDC, USDT, EURC, and PYUSD across 50+ currencies."
---

The off-ramp endpoint converts a specified stablecoin amount into fiat currency and initiates a bank transfer to the provided account. Once submitted, RemitFlex handles conversion, compliance checks, and settlement automatically. You can track progress via the returned `id` or by listening for `offramp.*` webhook events.

<Note>
  Your account must pass KYB (Know Your Business) verification before you can initiate live off-ramps. Test-mode requests work without KYB. Visit **Dashboard → Compliance → KYB** to submit your verification documents.
</Note>

## Endpoint

```
POST https://api.remitflex.com/v1/offramp
```

## Request Parameters

<ParamField body="amount" type="number" required>
  Amount of the source stablecoin to off-ramp. Must be a positive number. Fractional amounts are accepted up to the precision supported by the source currency (e.g., 6 decimal places for USDC).
</ParamField>

<ParamField body="source_currency" type="string" required>
  The stablecoin you are converting from. Accepted values: `USDC`, `USDT`, `EURC`, `PYUSD`.
</ParamField>

<ParamField body="destination_currency" type="string" required>
  The target fiat currency for the bank deposit, expressed as an [ISO 4217](https://www.iso.org/iso-4217-currency-codes.html) currency code (e.g., `EUR`, `USD`, `GBP`, `NGN`).
</ParamField>

<ParamField body="bank_account" type="object" required>
  Bank account details for the fiat deposit. All sub-fields apply to the receiving account.

  <Expandable title="bank_account fields">
    <ParamField body="account_holder_name" type="string" required>
      Legal name of the bank account holder exactly as it appears on the account. Used for beneficiary matching during settlement.
    </ParamField>

    <ParamField body="account_number" type="string" required>
      IBAN (for SEPA transfers) or local account number. For US ACH, provide the account number and pair it with `routing_number`.
    </ParamField>

    <ParamField body="routing_number" type="string">
      ABA routing number. **Required for US ACH transfers.** Not needed for SEPA or SWIFT transfers.
    </ParamField>

    <ParamField body="bank_name" type="string" required>
      Full name of the receiving bank (e.g., `"Deutsche Bank"`, `"Chase"`).
    </ParamField>

    <ParamField body="bank_country" type="string" required>
      Two-letter country code of the bank's domicile, per [ISO 3166-1 alpha-2](https://www.iso.org/iso-3166-country-codes.html) (e.g., `"DE"`, `"US"`, `"GB"`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="reference" type="string">
  An optional internal reference string you can attach to this off-ramp for your own reconciliation (e.g., a payout batch ID or invoice number). Maximum 255 characters. Returned as-is on the off-ramp object and included in webhook payloads.
</ParamField>

<ParamField body="metadata" type="object">
  An optional set of arbitrary key-value pairs you can attach to the off-ramp object. Keys and values must be strings. Useful for storing internal identifiers or context that you want returned in the response and webhook events.
</ParamField>

## Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the off-ramp, prefixed with `ofr_`. Use this to retrieve status or correlate with webhook events.
</ResponseField>

<ResponseField name="status" type="string">
  Current lifecycle state of the off-ramp. See the [status reference](/api-reference/offramp/status#status-values) for the full set of values and transitions.
</ResponseField>

<ResponseField name="amount" type="number">
  The stablecoin amount submitted for conversion, exactly as provided in the request.
</ResponseField>

<ResponseField name="source_currency" type="string">
  The stablecoin that was converted (e.g., `"USDC"`).
</ResponseField>

<ResponseField name="destination_currency" type="string">
  The ISO 4217 fiat currency code for the bank deposit (e.g., `"EUR"`).
</ResponseField>

<ResponseField name="converted_amount" type="number">
  The fiat amount that will be deposited into the bank account, after conversion and fees.
</ResponseField>

<ResponseField name="fee" type="number">
  The total fee charged for this off-ramp, denominated in the `source_currency`.
</ResponseField>

<ResponseField name="bank_account" type="object">
  Echo of the bank account details provided in the request.

  <Expandable title="bank_account fields">
    <ResponseField name="account_holder_name" type="string">
      Legal name of the bank account holder.
    </ResponseField>

    <ResponseField name="account_number" type="string">
      IBAN or local account number provided in the request.
    </ResponseField>

    <ResponseField name="routing_number" type="string">
      ABA routing number, if provided.
    </ResponseField>

    <ResponseField name="bank_name" type="string">
      Name of the receiving bank.
    </ResponseField>

    <ResponseField name="bank_country" type="string">
      ISO 3166-1 alpha-2 country code of the bank's domicile.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="reference" type="string">
  Your internal reference string, returned exactly as submitted.
</ResponseField>

<ResponseField name="estimated_settlement" type="string">
  ISO 8601 timestamp indicating when RemitFlex expects the funds to reach the destination bank. Actual settlement may vary based on the receiving bank's processing times.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the off-ramp was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the most recent status update on this off-ramp.
</ResponseField>

<ResponseField name="metadata" type="object">
  The arbitrary key-value pairs you attached to this off-ramp at creation time, returned as-is.
</ResponseField>

## Example Request & Response

<CodeGroup>

```bash curl
curl -X POST https://api.remitflex.com/v1/offramp \
  -H "Authorization: Bearer rf_live_xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 2500,
    "source_currency": "USDC",
    "destination_currency": "EUR",
    "bank_account": {
      "account_holder_name": "Acme Corp GmbH",
      "account_number": "DE89370400440532013000",
      "bank_name": "Deutsche Bank",
      "bank_country": "DE"
    },
    "reference": "Payout batch #42"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.remitflex.com/v1/offramp', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer rf_live_xxxx',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    amount: 2500,
    source_currency: 'USDC',
    destination_currency: 'EUR',
    bank_account: {
      account_holder_name: 'Acme Corp GmbH',
      account_number: 'DE89370400440532013000',
      bank_name: 'Deutsche Bank',
      bank_country: 'DE',
    },
    reference: 'Payout batch #42',
  }),
});

const offramp = await response.json();
console.log(offramp.id); // "ofr_01HX9N4R..."
```
