---
title: "POST /payments — Create Cross-Border Payment"
sidebarTitle: "Create Payment"
description: "Submit a new cross-border stablecoin payment to a recipient. Specify the source currency, corridor, and recipient banking details to initiate settlement."
---

The Create Payment endpoint initiates a cross-border stablecoin payment to a recipient bank account. You supply the source stablecoin amount, the destination currency, the corridor identifier, and your recipient's banking details — RemitFlex then orchestrates on-chain settlement and fiat delivery on your behalf. Optionally, you can lock in an exchange rate by attaching a `quote_id` from the [Conversions Quote](/api-reference/conversions/quote) endpoint before submitting.

## Request

<ParamField body="amount" type="number" required>
  Payment amount denominated in the source `currency`. Must be a positive number with up to 6 decimal places.
</ParamField>

<ParamField body="currency" type="string" required>
  Source stablecoin to send. Accepted values: `USDC`, `USDT`, `EURC`, `PYUSD`.
</ParamField>

<ParamField body="destination_currency" type="string" required>
  Target currency the recipient will receive. Accepts fiat ISO 4217 codes (e.g., `EUR`, `MXN`, `GBP`) or a supported stablecoin symbol.
</ParamField>

<ParamField body="corridor" type="string" required>
  Payment corridor identifier that defines the source-to-destination routing path (e.g., `US-EU`, `US-MX`). Corridors determine available currencies, cut-off times, and delivery SLAs.
</ParamField>

<ParamField body="recipient" type="object" required>
  Details of the individual or entity receiving the payment.

  <Expandable title="recipient fields">
    <ParamField body="name" type="string" required>
      Full legal name of the recipient as it appears on their bank account.
    </ParamField>

    <ParamField body="account_number" type="string" required>
      IBAN or local account number for the destination bank account.
    </ParamField>

    <ParamField body="bank_code" type="string" required>
      BIC/SWIFT code or local routing/sort code for the recipient's bank.
    </ParamField>

    <ParamField body="bank_country" type="string" required>
      ISO 3166-1 alpha-2 country code of the recipient's bank (e.g., `ES`, `MX`, `GB`).
    </ParamField>

    <ParamField body="address" type="object">
      Optional postal address for the recipient. Required on some corridors for compliance purposes.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="reference" type="string">
  Your internal payment reference, such as an invoice number or order ID. Maximum 255 characters. Passed through to the recipient bank where supported.
</ParamField>

<ParamField body="quote_id" type="string">
  ID of a pre-fetched conversion quote (prefix `qt_`). When provided, the locked exchange rate from that quote is applied. If omitted, RemitFlex applies the live market rate at the time of processing. Quotes expire after 30 seconds — see [Get a Quote](/api-reference/conversions/quote).
</ParamField>

<ParamField body="metadata" type="object">
  Arbitrary key-value pairs you can attach to the payment for your own tracking or reconciliation. Values must be strings. Not forwarded to the recipient or their bank.
</ParamField>

## Response

<ResponseField name="id" type="string">
  Unique payment identifier with prefix `pmt_`. Use this ID to retrieve, monitor, or cancel the payment.
</ResponseField>

<ResponseField name="status" type="string">
  Current lifecycle status of the payment. Possible values:

  | Value | Description |
  |---|---|
  | `pending` | Payment received and queued for processing. |
  | `processing` | On-chain settlement is in progress. |
  | `settled` | Stablecoin settlement confirmed on-chain. |
  | `delivered` | Funds delivered to the recipient's bank account. |
  | `failed` | Payment could not be completed. |
  | `cancelled` | Payment was cancelled before processing began. |
</ResponseField>

<ResponseField name="amount" type="number">
  The source payment amount you submitted.
</ResponseField>

<ResponseField name="currency" type="string">
  Source stablecoin used for the payment.
</ResponseField>

<ResponseField name="destination_currency" type="string">
  Target currency the recipient receives.
</ResponseField>

<ResponseField name="converted_amount" type="number">
  Amount delivered to the recipient in `destination_currency`, after applying the exchange rate and deducting fees.
</ResponseField>

<ResponseField name="exchange_rate" type="number">
  The applied conversion rate from `currency` to `destination_currency`.
</ResponseField>

<ResponseField name="fee" type="number">
  Total RemitFlex fee deducted from the payment, denominated in `currency`.
</ResponseField>

<ResponseField name="recipient" type="object">
  Recipient details associated with the payment.

  <Expandable title="recipient fields">
    <ResponseField name="name" type="string">
      Full legal name of the recipient.
    </ResponseField>

    <ResponseField name="account_number" type="string">
      Recipient's IBAN or local account number.
    </ResponseField>

    <ResponseField name="bank_country" type="string">
      ISO 3166-1 alpha-2 country code of the recipient's bank.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="reference" type="string">
  The internal reference you provided at creation, if any.
</ResponseField>

<ResponseField name="corridor" type="string">
  The corridor identifier used to route the payment.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the payment was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the most recent status update.
</ResponseField>

<ResponseField name="estimated_delivery" type="string">
  ISO 8601 timestamp of the projected delivery time to the recipient's bank account. This is an estimate and may vary based on corridor SLAs and banking cut-off times.
</ResponseField>

## Examples

<CodeGroup>

```bash curl
curl -X POST https://api.remitflex.com/v1/payments \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 500,
    "currency": "USDC",
    "destination_currency": "EUR",
    "corridor": "US-EU",
    "recipient": {
      "name": "Maria Garcia",
      "account_number": "ES9121000418401234567891",
      "bank_code": "CAIXESBBXXX",
      "bank_country": "ES"
    },
    "reference": "Invoice #INV-2024-001"
  }'
```

```javascript JavaScript
const response = await fetch("https://api.remitflex.com/v1/payments", {
  method: "POST",
  headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    amount: 500,
    currency: "USDC",
    destination_currency: "EUR",
    corridor: "US-EU",
    recipient: {
      name: "Maria Garcia",
      account_number: "ES9121000418401234567891",
      bank_code: "CAIXESBBXXX",
      bank_country: "ES",
    },
    reference: "Invoice #INV-2024-001",
  }),
});

const payment = await response.json();
console.log(payment.id); // pmt_01HX4N3RABCDEF...
```
