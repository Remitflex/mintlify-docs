---
title: "GET /payments/{id} — Retrieve Payment by ID"
sidebarTitle: "Get Payment"
description: "Fetch the full details and current status of a single payment by its unique ID. Use this to poll payment progress or verify final settlement."
---

The Get Payment endpoint retrieves the complete details of a specific payment using its unique identifier. You can use this endpoint to poll the current `status` of a payment, confirm delivery, or surface transaction details in your own dashboard. The payment object returned is identical in structure to the one returned when you [created the payment](/api-reference/payments/create).

## Path Parameters

<ParamField path="id" type="string" required>
  The unique payment identifier, prefixed with `pmt_` (e.g., `pmt_01HX4N3RABCDEF`). This is returned in the `id` field when you create a payment.
</ParamField>

## Response

<ResponseField name="id" type="string">
  Unique payment identifier with prefix `pmt_`.
</ResponseField>

<ResponseField name="status" type="string">
  Current lifecycle status of the payment. Possible values:

  | Value | Description |
  |---|---|
  | `pending` | Payment received and queued for processing. |
  | `processing` | On-chain settlement is in progress. |
  | `settled` | Stablecoin settlement confirmed on-chain. |
  | `delivered` | Funds delivered to the recipient's bank account. |
  | `failed` | Payment could not be completed. |
  | `cancelled` | Payment was cancelled before processing began. |
</ResponseField>

<ResponseField name="amount" type="number">
  The source payment amount originally submitted.
</ResponseField>

<ResponseField name="currency" type="string">
  Source stablecoin used for the payment.
</ResponseField>

<ResponseField name="destination_currency" type="string">
  Target currency the recipient receives.
</ResponseField>

<ResponseField name="converted_amount" type="number">
  Amount delivered to the recipient in `destination_currency`, after applying the exchange rate and deducting fees.
</ResponseField>

<ResponseField name="exchange_rate" type="number">
  The applied conversion rate from `currency` to `destination_currency`.
</ResponseField>

<ResponseField name="fee" type="number">
  Total RemitFlex fee deducted from the payment, denominated in `currency`.
</ResponseField>

<ResponseField name="recipient" type="object">
  Recipient details associated with the payment.

  <Expandable title="recipient fields">
    <ResponseField name="name" type="string">
      Full legal name of the recipient.
    </ResponseField>

    <ResponseField name="account_number" type="string">
      Recipient's IBAN or local account number.
    </ResponseField>

    <ResponseField name="bank_country" type="string">
      ISO 3166-1 alpha-2 country code of the recipient's bank.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="reference" type="string">
  The internal reference you provided at creation, if any.
</ResponseField>

<ResponseField name="corridor" type="string">
  The corridor identifier used to route the payment.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the payment was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the most recent status update.
</ResponseField>

<ResponseField name="estimated_delivery" type="string">
  ISO 8601 timestamp of the projected delivery time to the recipient's bank account.
</ResponseField>

## Examples

<CodeGroup>

```bash curl
curl https://api.remitflex.com/v1/payments/pmt_01HX4N3RABCDEF \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch(
  "https://api.remitflex.com/v1/payments/pmt_01HX4N3RABCDEF",
  {
    headers: {
      Authorization: "Bearer YOUR_API_KEY",
    },
  }
);

const payment = await response.json();
console.log(payment.status); // "delivered"
```

</CodeGroup>

## Error Responses

If the payment ID does not exist or does not belong to your account, the API returns a `404 Not Found` response.

<CodeGroup>

```json 404 Not Found
{
  "error": {
    "code": "NOT_FOUND",
    "message": "Payment pmt_01HX4N3RABCDEF not found."
  }
}
```

</CodeGroup>
