---
title: "POST /conversions/execute — Execute a Currency Conversion"
sidebarTitle: "Execute Conversion"
description: "Execute a currency conversion against a previously obtained quote. Locks in the quoted rate and immediately settles the converted amount to your RemitFlex balance."
---

The Execute Conversion endpoint consumes a quote obtained from [POST /conversions/quote](/api-reference/conversions/quote) and immediately performs the currency conversion at the locked rate. This is useful when you want to convert stablecoin balances within your RemitFlex account without initiating a payment — for example, to pre-fund a specific currency balance before a batch of payouts. You must execute the conversion within 30 seconds of requesting the quote, or the `quote_id` will have expired.

## Request

<ParamField body="quote_id" type="string" required>
  The unique quote identifier returned by [POST /conversions/quote](/api-reference/conversions/quote), prefixed with `qt_`. The quote must not be expired at the time of this request. Quotes are valid for 30 seconds from their `created_at` timestamp.
</ParamField>

## Response

<ResponseField name="conversion_id" type="string">
  Unique conversion identifier with prefix `con_`. Use this to reference the conversion in your records.
</ResponseField>

<ResponseField name="quote_id" type="string">
  The `qt_`-prefixed quote identifier that was consumed to execute this conversion.
</ResponseField>

<ResponseField name="status" type="string">
  Final status of the conversion. Possible values:

  | Value | Description |
  |---|---|
  | `completed` | Conversion executed successfully and funds are available in your balance. |
  | `failed` | Conversion could not be completed. No funds were moved. |
</ResponseField>

<ResponseField name="from_currency" type="string">
  The source currency that was converted from.
</ResponseField>

<ResponseField name="to_currency" type="string">
  The target currency that was converted into.
</ResponseField>

<ResponseField name="amount" type="number">
  The source amount that was converted, denominated in `from_currency`.
</ResponseField>

<ResponseField name="converted_amount" type="number">
  The amount credited to your balance in `to_currency` after the conversion and fee deduction.
</ResponseField>

<ResponseField name="exchange_rate" type="number">
  The exact exchange rate that was applied, as locked in the original quote.
</ResponseField>

<ResponseField name="fee" type="number">
  The RemitFlex fee charged for the conversion, denominated in `from_currency`.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the conversion was initiated.
</ResponseField>

<ResponseField name="completed_at" type="string">
  ISO 8601 timestamp of when the conversion was completed. `null` if `status` is `failed`.
</ResponseField>

## Examples

<CodeGroup>

```bash curl
curl -X POST https://api.remitflex.com/v1/conversions/execute \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "quote_id": "qt_01HX9M2PABCDEF"
  }'
```

```javascript JavaScript
const response = await fetch(
  "https://api.remitflex.com/v1/conversions/execute",
  {
    method: "POST",
    headers: {
      Authorization: "Bearer YOUR_API_KEY",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      quote_id: "qt_01HX9M2PABCDEF",
    }),
  }
);

const conversion = await response.json();

if (conversion.status === "completed") {
  console.log(
    `Converted ${conversion.amount} ${conversion.from_currency} → ` +
    `${conversion.converted_amount} ${conversion.to_currency}`
  );
}
```

</CodeGroup>

## Error Responses

If the `quote_id` has expired (more than 30 seconds have passed since the quote was created), the API returns a `422 Unprocessable Entity` error. You must request a fresh quote and retry.

<CodeGroup>

```json 422 Unprocessable Entity
{
  "error": {
    "code": "QUOTE_EXPIRED",
    "message": "Quote qt_01HX9M2PABCDEF has expired. Please request a new quote."
  }
}
```

</CodeGroup>

If the `quote_id` does not exist or was already consumed by a previous execution, the API returns a `404 Not Found` error.

<CodeGroup>

```json 404 Not Found
{
  "error": {
    "code": "NOT_FOUND",
    "message": "Quote qt_01HX9M2PABCDEF not found or has already been used."
  }
}
```

</CodeGroup>
